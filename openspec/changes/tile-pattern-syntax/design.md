## Context

SET/GET TILE PATTERN commands need syntax improvements to align with the sprite command patterns already established.

## Goals / Non-Goals

**Goals:**
- Align tile pattern syntax with sprite pattern commands

## Decisions

### Decision 1: Mirror SPRITE command patterns
Follow the same parameter structure as SET/GET SPRITE PATTERN for consistency.

## Risks / Trade-offs

- Syntax changes may break existing programs using the old syntax.
