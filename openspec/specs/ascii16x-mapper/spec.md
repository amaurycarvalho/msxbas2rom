## ADDED Requirements

### Requirement: ASCII16-X MegaROM mapper with 16KB pages and 12-bit bank registers

The system SHALL support compilation of MSX-BASIC programs into ROM images targeting the ASCII16-X MegaROM mapper. The ASCII16-X mapper extends ASCII16 with mirror address decoding and 12-bit bank number registers (lower 8 bits via D0-D7, upper 4 bits via A8-A11 of the write address).

For ROMs generated by msxbas2rom (≤4MB, ≤256 banks × 16KB), the upper 4 bank bits (A8-A11) SHALL always be 0. The kernel SHALL write bank numbers to base address 0x7000 exclusively (upper bank at 0x8000-0xBFFF), using the same dispatch-table-guided patching as ASCII16. 0x6000 is the hardware switch address for the lower bank (0x4000-0x7FFF) but is never written — the lower bank is permanently fixed to segment 0. Mirror addresses (0x2000-0x2FFF, 0xA000-0xAFFF, 0xE000-0xEFFF, 0x3000-0x3FFF, 0xB000-0xBFFF, 0xF000-0xFFFF) SHALL NOT be used.

The ASCII16-X mapper SHALL use the following memory layout:
- Page 1 (0x4000-0x7FFF, mirrored at 0xC000-0xFFFF): lower bank, permanently fixed to segment 0 (kernel code). 0x6000 is the hardware switch address but is never written.
- Page 2 (0x8000-0xBFFF, mirrored at 0x0000-0x3FFF): upper bank, switched via base address 0x7000, initial segment 0.

The kernel SHALL reuse the ASCII16 `fixAscii16Mapper()` patches exactly (7 patch points: 1 SeqReplace for MR_CHANGE_SGM with `push af; srl a; ld (0x7000),a; pop af; ret`, 1 ByteReplace + 3 NOPs for boot bugfix, 2 SeqReplace for OMSX autodetection at 0x77FF). Additionally, for ASCII16X mode only, the 14-byte `AB` signature check at `megarom_ascii8_bug_fix` SHALL be NOPped via dispatch table entry `DISP_ASCII16X_PATCH_BUGFIX_AB_CHECK` (index 223). The compiler SHALL remain unchanged for segment math.

The ASCII16-X autodetection signature `ASCII16X` SHALL be written at ROM offset 0x0010 for ASCII16X mode only. For this, the kernel header SHALL reserve 8 bytes (via `ds 8, 0x00`) between the `MSXB2R` custom signature and `INIT1`, shifting the entry point from 0x4010 to 0x4018. The `KERNEL_END_FILLER` SHALL absorb the size delta. The hardcoded `"LOADER"` kernel symbol address in `symbol_export_context.cpp` SHALL be updated from `"4010"` to `"4018"`.

ROM alignment SHALL pad to multiples of 8 pages (128KB). Maximum ROM size SHALL be 2048KB. Resource segment mapping SHALL use the 8KB-pair convention (`resourceSegment = pages.size() * 2`).

#### Scenario: Compile a program in ASCII16-X mode
- **WHEN** a valid MSX-BASIC program is compiled with `--ascii16x` flag
- **THEN** the output ROM file is generated with `[ASCII16X]` suffix
- **AND** the kernel is patched with 7 dispatch-table patches + the 8-byte signature at offset 0x0010
- **AND** the kernel's `MR_CHANGE_SGM` routine uses `push af; srl a; ld (0x7000),a; pop af; ret`
- **AND** all bank register writes target base address 0x7000 exclusively (lower bank at 0x4000-0x7FFF is fixed)
- **AND** the ROM is padded to a multiple of 128KB (8 pages)

#### Scenario: Verify ASCII16-X ROM kernel patching
- **WHEN** an ASCII16-X ROM is built
- **THEN** `MR_CHANGE_SGM` SHALL contain the byte sequence `F5 CB 3F 32 00 70 F1 C9` (push af / srl a / ld (0x7000),a / pop af / ret)
- **AND** 7 patch points SHALL be applied (same as ASCII16)
- **AND** the boot bugfix `ld (0x6800),a` SHALL be patched to `ld (0x7000),a`
- **AND** redundant bugfix writes SHALL be NOPped
- **AND** the 14-byte `AB` signature check SHALL be NOPped (`ld a,(0x8000); cp 0x41; jr nz; ld a,(0x8001); cp 0x42; jr nz` replaced with 0x00)
- **AND** the 4th and 5th OPENMSX writes SHALL target 0x77FF for mapper identification

#### Scenario: ASCII16-X signature present at ROM offset 0x0010
- **WHEN** an ASCII16-X ROM is built
- **THEN** the 8 bytes at ROM offset 0x0010-0x0017 SHALL be `41 53 43 49 49 31 36 58` ("ASCII16X")
- **AND** the ROM entry point pointer at offset 0x0002 SHALL point to 0x4018 (not 0x4010)
- **AND** INIT1 code SHALL begin at 0x4018

#### Scenario: ASCII16-X ROM size is valid
- **WHEN** an ASCII16-X ROM is generated
- **THEN** the ROM size SHALL be a multiple of 16KB pages
- **AND** the ROM size SHALL be between 128KB and 2048KB (after MegaROM filler padding)

#### Scenario: ASCII16-X uses base address 0x7000 only
- **WHEN** a bank register write occurs during ASCII16-X ROM execution
- **THEN** the write address SHALL be 0x7000 exclusively (upper bank)
- **AND** no mirror address SHALL be used for bank switching
- **AND** the lower bank (0x4000-0x7FFF) SHALL remain permanently fixed

#### Scenario: LOADER symbol address updated for all modes
- **WHEN** the kernel binary is regenerated via `make header`
- **THEN** `INIT1` SHALL be at 0x4018 (not 0x4010)
- **AND** the `"LOADER"` kernel symbol in `symbol_export_context.cpp` SHALL reference `"4018"`
- **AND** exported symbol files (.noi, .symbol, .omds, .cdb, .elf) SHALL contain `LOADER 4018H`
